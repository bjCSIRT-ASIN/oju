First Launch

Vue d’ensemble

Guide de vérification et configuration initiale après l’installation de Oju. Ces étapes garantissent le bon fonctionnement de la plateforme avant de commencer la surveillance de vos entités.

Vérifications système

Statut des services

# Vérifier que tous les conteneurs fonctionnent
docker-compose ps

# Résultat attendu : tous les services "Up"
#   Name                 State    Ports
# Oju_backend     Up       8000/tcp
# Oju_frontend    Up       3000/tcp
# Oju_nginx       Up       80/tcp, 443/tcp
# Oju_postgres    Up       5432/tcp
# Oju_redis       Up       6379/tcp
# Oju_celery_*    Up

Test de connectivité

Accès HTTPS :

# Tester l'accès à l'application
curl -I https://votre-domaine.com

# Réponse attendue : HTTP/2 200
# Avec headers sécurité (HSTS, etc.)

Vérification certificat SSL :

# Contrôler la validité du certificat
openssl s_client -connect votre-domaine.com:443 -servername votre-domaine.com

# Vérifier la date d'expiration
echo | openssl s_client -connect votre-domaine.com:443 -servername votre-domaine.com 2>/dev/null | openssl x509 -noout -dates

Accès aux interfaces

Interface principale

URL : https://votre-domaine.com

Première connexion : 1. Ouvrir le navigateur sur votre domaine 2. Vérifier l’absence d’erreur SSL 3. Page de connexion doit s’afficher

Interface d’administration Django

URL : https://votre-domaine.com/admin

Connexion administrateur : 1. Utiliser les identifiants créés lors de l’installation 2. Interface Django admin doit être accessible 3. Menu de gestion visible

Configuration initiale

Paramètres de site Django

Dans l’interface admin Django :

  1. Sites → Sites → Modifier l’entrée par défaut

  2. Nom de domaine : votre-domaine.com

  3. Nom affiché : Oju - Monitoring Cybersécurité

  4. Sauvegarder les modifications

Création d’utilisateurs supplémentaires

Utilisateurs administrateurs : 1. Authentification et autorisation → Utilisateurs 2. Ajouter utilisateur → Remplir les informations 3. Cocher Statut équipe et Statut superutilisateur 4. Assigner les permissions appropriées

Utilisateurs standard : 1. Créer sans privilèges administrateur 2. Configurer email pour récupération de compte 3. Assigner aux groupes selon les besoins

Test des fonctionnalités principales

Dashboard : 1. Accéder au Dashboard depuis l’interface principale 2. Vérifier l’affichage des widgets (peuvent être vides au démarrage) 3. Contrôler l’absence d’erreurs de permissions

Gestion des entités : 1. Naviguer vers Entities 2. Tester l’ajout d’une entité de test 3. Vérifier la sauvegarde et l’affichage

Configuration de surveillance

Ajout de votre première entité

Création d’organisation test : 1. Entities → Add entity 2. Nom : Test Organization 3. Description : Entité de test pour validation 4. Assigner des focal points si disponibles 5. Sauvegarder

Ajout de plateformes de test

Création de sites à surveiller : 1. Depuis la vue détaillée de l’entité 2. Add platform → URL de test 3. Statut : Actif 4. Vérifier l’association à l’entité

Test du système d’alertes

Vérification du monitoring : 1. Attendre quelques minutes pour les premiers scans 2. Vérifier l’apparition d’alertes dans Alerts 3. Tester la modification de statut d’une alerte 4. Contrôler les logs système

Surveillance des performances

Vérification des logs

# Logs généraux
docker-compose logs --tail=50

# Logs par service
docker-compose logs backend
docker-compose logs celery_worker

# Suivi temps réel si nécessaire
docker-compose logs -f backend

Monitoring des ressources

# Utilisation des ressources
docker stats

# Espace disque
docker system df

# Volumes utilisés
docker volume ls

Configuration de la sauvegarde

Sauvegarde configuration

# Sauvegarde fichiers critiques
tar -czf Oju-config-$(date +%Y%m%d).tar.gz .env docker-compose.yml

# Sauvegarde certificats SSL
tar -czf Oju-ssl-$(date +%Y%m%d).tar.gz nginx/ssl/

Sauvegarde base de données

# Export PostgreSQL
docker-compose exec postgres pg_dump -U postgres Oju_db > Oju-db-$(date +%Y%m%d).sql

# Test de restauration (optionnel)
cat Oju-db-YYYYMMDD.sql | docker-compose exec -T postgres psql -U postgres Oju_db

Configuration surveillance avancée

Notifications email (optionnel)

Si configuré dans le .env :

# Test envoi email depuis Django
docker-compose exec backend python manage.py shell -c "
from django.core.mail import send_mail
send_mail('Test Oju', 'Email de test', 'Oju@votre-domaine.com', ['admin@votre-domaine.com'])
"

Planification des tâches

Vérifier que Celery Beat fonctionne :

# Logs du scheduler
docker-compose logs celery_beat

# Vérifier l'exécution des tâches
docker-compose logs celery_worker | grep -i "task"

Résolution de problèmes

Services qui ne démarrent pas

# Identifier le service problématique
docker-compose ps

# Consulter les logs détaillés
docker-compose logs service_name

# Redémarrage ciblé
docker-compose restart service_name

Problèmes de connexion

Erreur SSL : 1. Vérifier la configuration DNS 2. Contrôler les certificats : ./update-certs.sh 3. Vérifier les logs Nginx : docker-compose logs nginx

Base de données inaccessible :

# Test de connectivité PostgreSQL
docker-compose exec postgres pg_isready -U postgres

# Test de connexion Django
docker-compose exec backend python manage.py check --database default

Performance dégradée

# Vérifier les ressources
docker stats

# Nettoyer si nécessaire
docker system prune

# Redémarrer tous les services
docker-compose restart

Étapes suivantes

  1. Créer des comptes pour votre équipe

  3. Documenter vos processus de réponse aux incidents

Configuration de production

  1. Planifier les créneaux de maintenance

  2. Configurer la surveillance des services

  3. Établir une procédure de sauvegarde régulière

  1. Surveiller les mises à jour Oju

  2. Planifier l’ajout de nouvelles entités

  3. Ajuster la configuration selon l’usage

Validation finale

✅ Tous les services Docker fonctionnent ✅ Accès HTTPS sans erreur SSL ✅ Interface admin Django accessible ✅ Dashboard principal opérationnel ✅ Première entité créée avec succès ✅ Système d’alertes fonctionnel ✅ Sauvegarde initiale effectuée

Oju est maintenant opérationnel et prêt pour la surveillance de vos entités en production !
