System Administration

Overview

System Administration provides centralized management of Cerberus core settings and email configurations. This section covers system-wide parameters that affect monitoring behavior, network connectivity, and notification capabilities.

System Configuration

General Settings

Core System Parameters:

Email Configuration: * email - Primary system email address for notifications * Purpose: Default sender address for system notifications * Format: Valid email address format required

Network Settings: * proxy - List of proxy servers for HTTP requests * Purpose: Route monitoring traffic through specified proxies * Format: Array of proxy URLs (e.g., ["http://proxy1:8080", "http://proxy2:8080"]) * Validation: Automatically removes duplicates

DNS Configuration: * dns_server - List of DNS servers for domain resolution * Purpose: Custom DNS servers for domain lookups and validation * Format: Array of IP addresses (e.g., ["8.8.8.8", "1.1.1.1"]) * Validation: Automatically removes duplicates

Scanning Parameters: * scan_frequency - Interval between monitoring scans * Purpose: Controls how often platforms are monitored * Format: Numeric value (minutes/hours based on system configuration)

  • user_agent - HTTP User-Agent string for web requests

  • Purpose: Identifies Cerberus in HTTP requests to monitored platforms

  • Format: Standard User-Agent string format

  • max_worker - Maximum number of concurrent scanning workers

  • Purpose: Controls system resource usage and scanning parallelism

  • Format: Positive integer value

Alert Settings: * receive_alert - Global alert reception toggle * Purpose: Enable/disable system-wide alert generation * Format: Boolean (true/false)

Toggle Controls

Proxy Management:

use_proxy: * Controls whether monitoring uses configured proxy servers * Toggle action automatically tests proxy connectivity when enabled * System logs proxy activation/deactivation events

use_host_on_proxy_fail: * Determines fallback behavior when proxy connections fail * When enabled: Falls back to direct connection if proxy unavailable * When disabled: Monitoring fails if proxy is unreachable

receive_alert: * Master switch for alert generation across the system * When disabled: No alerts are generated regardless of detection * When enabled: Alerts generated based on monitoring results

Configuration Management

Updating Settings: 1. Navigate to System Configuration page 2. Modify desired parameters 3. System validates input format and values 4. Changes are applied immediately with system logging 5. Cache invalidation occurs for DNS settings

Validation Rules: * Only specified fields can be modified * Lists (proxy, dns_server) must be valid arrays * Email addresses must follow standard format * Numeric values must be positive integers * Boolean toggles accept true/false values

All configuration changes are logged in the system audit trail with user identification and timestamp.

Email Configuration

SMTP Settings

Server Configuration:

smtp_server: * SMTP server hostname or IP address * Purpose: Outgoing mail server for notifications * Example: "smtp.gmail.com", "mail.company.com"

smtp_port: * SMTP server port number * Purpose: Network port for SMTP connection * Common values: 25, 465 (SSL), 587 (TLS)

Security Settings:

use_tls: * Enable STARTTLS encryption for SMTP connection * Purpose: Secure email transmission using TLS * Typically used with port 587

use_ssl: * Enable SSL/TLS encryption from connection start * Purpose: Secure email transmission using SSL * Typically used with port 465

TLS and SSL are mutually exclusive. Configure only one encryption method based on your SMTP server requirements.

Authentication:

email_host: * Username for SMTP authentication * Purpose: Account credentials for sending emails * Usually an email address or username

email_password: * Password for SMTP authentication * Purpose: Account credentials for sending emails * Stored securely in the system

Sender Information:

default_sender_name: * Display name for outgoing emails * Purpose: Human-readable sender identification * Example: "Cerberus Security Monitor"

is_active: * Master switch for email functionality * Purpose: Enable/disable all email notifications * When disabled: No emails are sent regardless of alerts

Email Configuration Management

Connection Testing: * Automatic SMTP connection test when saving configuration * Validates server connectivity, authentication, and security settings * Configuration is rejected if connection test fails * Test includes DNS resolution, port connectivity, and authentication

Toggle Status: * Email system can be enabled/disabled without changing configuration * Enabling inactive configuration triggers automatic connection test * System prevents activation if connection test fails * Status changes are logged for audit purposes

Rate Limiting: * Email configuration updates limited to 2 attempts per 60 seconds * Prevents excessive server testing and potential account lockouts * Limit applies to configuration updates, not toggle operations * Rate limit reset after timeout period

Email configuration changes trigger immediate connection testing. Ensure SMTP credentials and server settings are correct to avoid account lockouts.

Configuration Process

Setting Up Email: 1. Navigate to Email Configuration page 2. Enter SMTP server details (hostname, port) 3. Select appropriate encryption method (TLS or SSL) 4. Provide authentication credentials 5. Set sender name for outgoing emails 6. Save configuration (triggers automatic testing) 7. Activate email system using toggle if test successful

Troubleshooting Connection Issues:

Common Problems: * DNS errors: Verify SMTP server hostname * Connection timeout: Check port number and firewall settings * Authentication failed: Verify username/password credentials * Connection refused: Confirm server availability and port accessibility

Testing Process: * System resolves SMTP server hostname * Establishes connection on specified port * Negotiates encryption if configured * Performs authentication with provided credentials * Reports success or specific error message

Security Considerations

System Security

Network Configuration: * Proxy settings can expose traffic to third-party servers * DNS server changes affect all domain resolution * Monitor proxy and DNS server trustworthiness

Access Control: * Configuration changes require appropriate permissions * All modifications are logged with user identification * Critical settings should be reviewed before changes

Email Security

Credential Protection: * SMTP passwords stored securely in database * Use dedicated email accounts for system notifications * Consider application-specific passwords for enhanced security

Encryption Best Practices: * Use TLS/SSL encryption for all SMTP connections * Verify certificate validity for SMTP servers * Prefer TLS (port 587) over legacy unencrypted connections

Operational Guidelines

System Maintenance

Regular Review: * Verify proxy server availability and performance * Monitor DNS server responsiveness * Review scanning frequency based on system load * Adjust worker count based on resource utilization

Change Management: * Test configuration changes in non-production environments * Document configuration changes for audit purposes * Monitor system behavior after configuration updates * Maintain backup of working configurations

Email Management

Email Monitoring: * Verify email delivery to intended recipients * Monitor SMTP server performance and availability * Review email logs for delivery issues * Test email functionality periodically

Best Practices: * Use dedicated email accounts for system notifications * Configure appropriate sender names for clarity * Monitor rate limiting to avoid configuration lockouts * Maintain current SMTP credentials

Troubleshooting

System Configuration Issues

Proxy Problems: * Test proxy connectivity independently * Verify proxy authentication if required * Check proxy server capacity and performance * Review firewall rules for proxy traffic

DNS Issues: * Test DNS server responsiveness * Verify DNS server accessibility from system * Check DNS resolution for monitored domains * Consider DNS server redundancy

Email Configuration Issues

Connection Failures: * Verify SMTP server hostname and port * Test network connectivity to SMTP server * Check firewall rules for SMTP traffic * Confirm SMTP server supports configured encryption

Authentication Problems: * Verify username and password accuracy * Check for account lockouts or restrictions * Confirm account has SMTP sending permissions * Consider application-specific passwords

Rate Limiting: * Wait for rate limit timeout before retry * Verify configuration accuracy before multiple attempts * Use toggle function for status changes (not rate limited) * Plan configuration changes to minimize test attempts

System Administration provides essential control over Cerberus monitoring behavior and notification capabilities, ensuring optimal performance and reliable alert delivery.
