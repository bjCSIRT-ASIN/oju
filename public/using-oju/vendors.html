Vendor Integrations

Overview

Vendor Integrations enable Oju to connect with external security services and manage antivirus vendor relationships. Configure automated ticket creation, threat intelligence, data synchronization, and maintain comprehensive vendor contact information.

Integration Categories

Antivirus Vendors

Local Vendor Database: * Vendor contact information management * Relationship tracking * Communication coordination * Alert correlation with vendor data

Security Tools Integration

External Services: * RTIR - Incident response ticket creation * VirusTotal - Malware and threat detection * Cerebrate - Data synchronization and management

Integration Benefits: * Automated workflow coordination * Enhanced threat intelligence * Streamlined incident response * Centralized data management

Antivirus Vendor Management

Vendor Information

Vendor Details: * Name - Antivirus vendor name * Contact - Primary contact information * Comments - Additional notes and details * Timestamps - Creation and modification dates

Managing Vendors

Adding Vendors: . Navigate to Vendor Integrations . Click Add Vendor . Enter vendor information: * Vendor name * Contact details * Relevant comments . Save to create vendor record

Updating Vendors: . Select existing vendor . Modify vendor information . Update contact details . Save changes

Vendor Operations: * Create new vendor entries * Update contact information * Remove outdated vendors * Track vendor relationships

Vendor Usage

Alert Integration: * VirusTotal results reference vendor database * Vendor contact information included in alerts * Automated vendor notification capabilities * Incident coordination support

RTIR Integration

Configuration Setup

Connection Details: * URL - RTIR server address (https://rtir.example.com) * Username - Authentication username * Password - Authentication password * Connection Testing - Automatic validation

Setup Process: . Navigate to Integrations > RTIR . Enter server connection details . Test connection validity . Activate integration . Configure ticket creation settings

Ticket Creation

Automatic Ticket Generation: * Alert-triggered ticket creation * Detailed incident information * Stakeholder notification * Priority assignment based on alert type

Ticket Content: * Subject - Alert type and affected platform * Description - Detailed alert information * Requestors - Entity focal points * Priority - Based on alert severity * Custom Fields - IP addresses, domains, CVE IDs

RTIR Operations

Status Management: * Enable/disable RTIR integration * Connection validation on activation * Automatic retry on failures * Error logging and notification

Ticket Workflow: * Oju alert generation * Automatic RTIR ticket creation * Stakeholder notification * Incident tracking coordination

VirusTotal Integration

Configuration

API Setup: * API Key - VirusTotal authentication key * Scan Frequency - Automated scan interval * Connection Validation - API key verification

Setup Steps: . Navigate to Integrations > VirusTotal . Enter valid API key . Configure scan frequency . Test API connectivity . Activate integration

Scanning Operations

Automated Scanning: * Periodic platform URL analysis * Malware detection alerts * Vendor analysis results * Threat intelligence correlation

Scan Results: * Malicious URL detection * Vendor-specific findings * Alert generation for threats * Detailed threat analysis

Alert Integration: * Automatic alert creation for threats * Vendor information inclusion * Stakeholder notification * Incident response coordination

Cerebrate Integration

Configuration Setup

Connection Parameters: * URL - Cerebrate server address * API Key - Authentication token * Refresh Frequency - Data synchronization interval

Setup Process: . Navigate to Integrations > Cerebrate . Configure server connection . Set synchronization frequency . Test connectivity . Activate integration

Data Synchronization

Synchronized Data: * Organizations - Entity information * Individuals - Focal point contacts * Tags - Functional classifications * Relationships - Entity-contact associations

Sync Process: * Automatic Sync - Scheduled synchronization * Manual Refresh - On-demand data update * Conflict Resolution - Data precedence rules * Status Tracking - Synchronization logging

Sync Operations

Data Management: * Create new entities from Cerebrate * Update existing contact information * Manage focal point relationships * Handle deactivated contacts

Manual Refresh: . Navigate to Integrations > Cerebrate . Click Manual Refresh . Monitor synchronization progress . Review sync statistics . Check logs for any issues

Integration Management

Status Control

Integration Status: * Active - Integration operational * Inactive - Integration disabled * Configuration - Setup in progress

Status Management: . Select integration service . Click Toggle Status . Confirm configuration validity . Monitor activation success . Review operational status

Connection Testing

Validation Process: * Automatic connection testing on activation * Credential verification * Service availability confirmation * Error reporting and logging

Test Results: * Success - Integration ready for use * Failure - Configuration issues identified * Error Details - Specific problem information * Resolution Guidance - Troubleshooting suggestions

Operational Workflows

Daily Operations

Integration Monitoring: . Check integration status . Review sync statistics . Monitor ticket creation . Verify scan operations

Maintenance Tasks: . Update vendor contact information . Review integration logs . Test connection validity . Update API keys as needed

Incident Response

Automated Workflows: . Alert detection triggers integrations . RTIR tickets created automatically . VirusTotal analysis performed . Vendor information referenced . Stakeholders notified

Manual Coordination: . Access vendor contact information . Create additional RTIR tickets . Perform manual VirusTotal scans . Coordinate response efforts

Troubleshooting

Common Issues

RTIR Problems: * Verify server URL accessibility * Check authentication credentials * Confirm network connectivity * Review firewall settings

VirusTotal Issues: * Validate API key accuracy * Check rate limiting status * Verify internet connectivity * Monitor quota usage

Cerebrate Problems: * Test API key validity * Verify server accessibility * Check synchronization logs * Review data conflicts

Best Practices

Integration Maintenance: * Regular connection testing * Periodic credential updates * Monitor integration logs * Update configurations as needed

Security Considerations: * Secure credential storage * API key rotation policies * Network security compliance * Access logging and monitoring

Security and Compliance

Data Protection

Secure Handling: * Encrypted credential storage * Secure API communications * Data transmission protection * Access audit logging

Privacy Compliance: * Personal data protection * Contact information security * Integration data handling * Retention policy compliance

Access Control

Permission Requirements: * Integration view permissions * Configuration edit permissions * Status toggle permissions * Administrative oversight

Audit Trail: * All configuration changes logged * Integration usage tracking * Status change monitoring * Error event recording

Vendor Integrations enhance Oju capabilities by connecting external security services and maintaining comprehensive vendor relationships for effective incident response and threat management.
